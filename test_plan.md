# test_plan.md

## テスト方針

ゲームロジックを中心に自動テストする。

## 単体テスト対象

### hash

- 同じ文字列は同じhashになる
- 異なる文字列は異なるhashになりやすい

### character

- 同じバーコードから同じステータスが生成される
- HPが80〜200に収まる
- ちからが10〜50に収まる
- ぼうぎょが5〜40に収まる
- すばやさが5〜35に収まる

### battle

- 通常攻撃でHPが減る
- ダメージは最低1
- ためる後の攻撃は強くなる
- まもる中は被ダメージが減る
- まもるは必殺に対して通常攻撃より強く効く
- 必殺は命中時に大ダメージ
- HPが0以下になったらwinnerが設定される

### localBattle

- プレイヤー1から選択が始まる
- プレイヤー1の選択後、コマンド名を公開せずにプレイヤー2へ進む
- 両者の選択後にラウンドが解決される
- すばやさ順に行動する
- まもるは同じラウンド内の攻撃を軽減する
- HPが0以下になったらwinnerが設定される

### network / Firebase同期

- 部屋作成時に部屋IDが生成される
- 満員の部屋には参加できない
- Firebase上の部屋データを型安全にパースできる
- 不正な部屋データや欠けた部屋データを弾ける
- キャラクター準備状態を書き込める
- コマンド選択状態を書き込める
- ホストは両者のコマンドが揃うまでラウンドを解決しない
- ゲストは `battle` 更新を受け取るまでHPやログを確定更新しない
- 切断時に接続状態が更新される

## 手動テスト

- iPhone Safari
- Android Chrome
- iPad Safari
- PC Chrome

## 通信対戦の手動テスト

- PC Chromeの別タブ同士で接続する
- PC ChromeとスマホChromeで同一ネットワーク接続する
- ホストが部屋を作成し、ゲストが部屋IDで参加する
- 両者がバーコードを入力し、キャラクターが同期される
- 片方がコマンド選択済みでも、相手側にコマンド内容が表示されない
- 両者が選択後、同じHP・ログ・勝敗が表示される
- 片方のタブを閉じたときに切断表示になる
